# pi-todowrite

[Pi](https://pi.dev)용 할 일 목록 관리 확장 프로그램. 다단계 작업 전에 자동으로
구조화된 할 일 목록을 생성하고 TUI에 표시합니다. Pi 자동 압축 이후에도 에이전트가
작업을 계속 이어가도록 유지합니다.

## 기능

- LLM이 호출하는 `todowrite` 도구를 등록하여 구조화된 작업 목록을 관리
- 시스템 프롬프트에 동작 규칙을 주입하여, 다단계 작업(2개 이상 파일, 3단계 이상,
  또는 위임/교차 작업) 전에 반드시 할 일 목록을 만들고 CHECK → EXECUTE → UPDATE
  워크플로 루프를 따르도록 강제
- TUI 위젯에 현재 할 일 목록을 실시간으로 표시
- **컴팩트 모드** (기본값): 미완료 항목만 표시하여 화면 공간 절약 — 완료된 항목은
  진행 요약(`done/total │ remaining`) 뒤로 숨겨짐
- **전체 모드**: 완료 항목을 포함한 모든 작업 표시
- **압축 후 자동 재개**: Pi가 자동 압축(임계치 또는 오버플로)을 실행하고 미완료
  할 일이 남아 있으면, 1회성 사용자 메시지를 보내 에이전트가 하던 작업을 이어가도록
  함 — 수동 "계속" 입력 불필요
- 할 일 목록을 세션 파일에 저장하여 압축과 세션 재개 시 유지

## 설치

```bash
pi install git:github.com/giocom/pi-todowrite
```

또는 직접 실행(테스트용):

```bash
pi -e ./src/index.ts
```

## 명령어

```
/todowrite                  # 현재 할 일 목록 표시
/todowrite status           # 위와 동일
/todowrite compact          # TUI 위젯: 미완료 항목만 (기본값)
/todowrite full             # TUI 위젯: 전체 항목
/todowrite toggle           # TUI 위젯 표시/숨김
/todowrite reset            # 모든 할 일 삭제
/todowrite autoresume       # 자동 재개 상태 확인
/todowrite autoresume on    # 압축 후 자동 재개 활성화 (기본값)
/todowrite autoresume off   # 자동 재개 비활성화
```

## 압축 후 자동 재개

Pi 내장 압축은 컨텍스트 윈도우가 가득 차면 실행됩니다. 압축 후 에이전트는 멈추고
사용자 입력을 기다립니다 — 작업이 아직 진행 중이더라도요. 이 확장 프로그램은
`session_compact` 이벤트를 감지하고, 미완료 할 일이 남아 있으면 1회성
`Continue with the current task.` 사용자 메시지를 보내 에이전트가 하던 작업을
이어가도록 합니다.

### 재개가 동작하는 경우

- **임계치 압축** (컨텍스트가 설정 임계치 초과) → 재개
- **오버플로 압축** (LLM이 컨텍스트 오버플로 발생) Pi가 자동 재시도하지 않는 경우 → 재개

### 재개가 동작하지 않는 경우

- **수동 `/compact`** — 사용자가 의도적으로 멈춤
- **`willRetry: true`인 오버플로** — Pi가 이미 중단된 턴을 재시도함
- **미완료 할 일 없음** — 재개할 작업이 없음
- **에이전트가 이미 바쁨** — 사용자(또는 다른 확장)가 이미 턴을 시작함
- **디바운스** — 에이전트 턴당 1회만 재개하여 압축-재개 루프 방지

`/todowrite autoresume on|off`로 전환 가능. 기본값은 켜짐.
